### 1. Python 基础回顾

- **缩进敏感**：Python 靠缩进控制代码层级，同一代码块必须上下对齐，与 C/C++ 的大括号不同
- **函数定义**：若定义空函数暂不实现逻辑，需使用 `pass` 语句占位，否则会报错
- **类与对象**：
    - 使用 `class` 定义，`__init__` 进行初始化（类似构造函数）
    - 成员变量在 `__init__` 中通过 `self` 定义，成员方法第一个参数必须是 `self`

### 2. PyTorch Tensor 核心概念

- **引入原因**：Python 列表数据类型不统一且无法管理 Shape；NumPy 无法计算模型梯度，不支持深度学习优化
- **基本属性**：
    - `shape`：张量的形状，决定能否进行矩阵乘法等运算
    - `dtype`：数据类型，多为浮点数（如 32 位、16 位），精度影响硬件要求和计算效率
    - `device`：数据所在的设备（CPU 或 GPU），运算前需确保数据在同一设备上
    - `requires_grad`：是否记录梯度，模型参数更新时需开启

### 3. Tensor 常见操作

- **创建方式**：`torch.tensor()`、`torch.zeros/ones()`、`torch.rand()`（随机初始化权重）
- **形状变换**：
    - `reshape(-1)`：展平张量
    - `transpose()`：转置操作
    - `unsqueeze/squeeze`：增加或减少维度（如增加 batch 维度）
- **运算与聚合**：
    - 矩阵乘法使用 `@` 符号，对应位置相乘使用 `*`
    - `broadcasting`：广播机制，允许不同形状的张量进行加法运算
    - `torch.cat`：串联张量，常用于拼接特征

### 4. 机器学习与深度学习基础

- **任务类型**：回归（预测）、分类、排序（推荐系统核心）、序列预测、生成
- **数据集划分**：
    - **训练集**：用于学习模型参数
    - **验证集**：伴随训练过程，用于监控过拟合、实现早停（Early Stopping）
    - **测试集**：训练结束后评估最终效果，严禁在训练中使用（防止数据泄露）
- **过拟合与欠拟合**：
    - **过拟合**：训练集表现好，验证/测试集表现差，说明学到了过多噪声细节
    - **欠拟合**：训练集和验证集损失均持续下降，未达到最优
- **经典模型**：线性回归、决策树、MLP（多层感知机）、CNN（卷积神经网络）、RNN/LSTM（循环神经网络）、Transformer（注意力机制）

### 5. 模型训练流程

- **DataLoader**：将数据集组织成可迭代对象，支持 `batch_size` 分批加载和 `shuffle` 打乱顺序
- **训练循环**：
    1. 从 DataLoader 取一个 batch
    2. 模型前向传播计算预测值
    3. 计算 Loss（如交叉熵）
    4. `optimizer.zero_grad()` 清空梯度
    5. `loss.backward()` 反向传播
    6. `optimizer.step()` 更新参数
